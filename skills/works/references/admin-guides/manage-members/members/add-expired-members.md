# 기한 만료 구성원 비밀번호 재생성

관리자가 구성원을 추가한 후, 구성원이 전달받은 계정 정보로 로그인하지 않은 채 30일이 경과하면 가입 기한이 만료됩니다.

기한이 만료된 구성원을 다시 추가하려면 관리자가 로그인에 필요한 정보를 구성원에게 다시 전달해야 합니다.

구성원 추가 시 [비밀번호를 생성](/ko/admin-guides/manage-members/members/create-password/)했던 방법에 따라 다시 추가하는 방법이 다릅니다.

# 비밀번호를 생성해서 전달해 추가한 경우

구성원 추가 시 비밀번호를 자동 생성 또는 관리자가 생성한 경우, 30일 내에 해당 구성원이 전달받은 정보로 로그인하지 않았다면, 초대한 구성원의 계정 상태는 '미접속(기한만료)'가 되고, 전달받은 초기 비밀번호로는 더 이상 로그인할 수 없게 됩니다.

구성원을 다시 추가하기 위해서는 관리자가 초기 비밀번호를 재설정해서 전달해야 합니다.

1. Admin 왼쪽 메뉴에서 '구성원'을 선택해 메뉴를 펼친 후 '구성원'을 눌러 '구성원' 화면으로 이동합니다. 모바일에서는 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2024/04/icon-admin-mobile-menu.png) 버튼을 탭하면 메뉴가 나타납니다.
2. 구성원 목록에서 재설정이 필요한 구성원 '계정 상태'에 표시된 '비밀번호 재설정'을 누릅니다.
3. '비밀번호 재설정' 창에서 비밀번호 설정 옵션을 선택합니다.
   - [자동으로 생성](/ko/admin-guides/manage-members/members/create-password/#i): 시스템에서 자동으로 임시 비밀번호를 생성
   - [관리자가 등록](/ko/admin-guides/manage-members/members/create-password/#i-2): 관리자가 직접 임시 비밀번호를 등록
4. '다음'을 선택하여 초기 비밀번호를 재설정합니다.
5. '인쇄하기' 또는 '메일 보내기'를 눌러 재설정된 비밀번호를 구성원에게 전달합니다.

# 구성원이 비밀번호를 생성하는 방법으로 추가한 경우

구성원 추가 시 비밀번호를 구성원이 직접 생성하는 방법을 선택한 경우, 30일 내에 해당 구성원이 초대 메일 내 비밀번호 생성 링크로 비밀번호를 설정해 로그인하지 않았다면, 초대한 구성원의 계정 상태는 '미접속(기한만료)'가 되고, 받은 초대 메일로는 더 이상 비밀번호를 생성해 로그인할 수 없습니다.

구성원을 다시 추가하기 위해서는 관리자가 초대 메일을 다시 보내야 합니다.

1. Admin 왼쪽 메뉴에서 '구성원'을 선택해 메뉴를 펼친 후 '구성원'을 눌러 '구성원' 화면으로 이동합니다. 모바일에서는 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2024/04/icon-admin-mobile-menu.png) 버튼을 탭하면 메뉴가 나타납니다.
2. 구성원 목록에서 재발송이 필요한 구성원의 오른쪽 '계정 상태' 항목에 표시된 '초대메일 재발송'을 누릅니다.
3. '초대메일 재발송' 창에 구성원의 메일 주소를 입력합니다.
4. 입력한 메일 주소를 구성원의 개인 이메일로 저장할지 선택 후, '보내기'를 선택해 초대 메일을 재발송합니다.

공공용 네이버웍스에서는 일부 기능이 제한됩니다. 자세한 내용은 [네이버웍스 기능 소개서](https://gov-naverworks.com/naver-works-function-list/)를 확인해주세요.
