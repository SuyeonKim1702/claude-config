# 설정

이 기능은 웍스 결재 상품 신청 후 사용 가능합니다.

  
결재 서비스의 각종 화면에 적용되는 설정을 회사 환경에 알맞게 변경할 수 있습니다. 또한 제공되는 템플릿을 사용하여 외부 문서를 손쉽게 불러올 수 있습니다.

# 결재 설정

결재 업무의 선결재, 순차 결재, 지정 반려, 보류, 서명 기능을 사용하거나 사용할 수 없도록 설정할 수 있습니다.

결재 설정 화면은 다음과 같이 구성되어 있습니다.

| 영역 | 설명 |
| --- | --- |
| ① 선결재 | 결재선에서 지정한 결재 순서와 상관 없이 결재 예정 문서를 선결재할 수 있도록 설정 |
| ④ 지정 반려 | 승인한 결재자 중 한 명을 지정하여 결재 승인을 취소, 해당 결재자 이후 결재자의 승인을 다시 받도록 설정  - 결재 순서는 변경되지 않으며, 선결재 사용 시 지정 반려 불가 |
| ⑤ 보류 | 문서 결재를 보류하여 결재 진행을 일시 중지할 수 있도록 설정  - 선결재 사용 시, 보류한 결재자 외의 다른 결재자는 결재 가능 |
| ⑦ 결재자 비 연속 중복 | 연속되지 않은 결재자의 중복 여부 설정(관리자의 결재선 설정에만 적용)  - '허용'으로 설정 시 연속되지 않은 순서에 같은 결재자 중복 설정 허용 - '허용 안 함'으로 설정 시 같은 결재자를 한 번만 설정할 수 있도록 처리하며(병렬 내 순차 포함) 및 중복 결재자의 삭제 방식 선택   - 첫번째 순서 남기고 삭제/마지막 순서 남기고 삭제: 중복 결재자를 첫번째 순서 또는 마지막 순서만 남기고 모두 삭제함 - 연속된 순서에 동일 결재자가 여러 번 중복으로 설정된 경우, 중복 결재자 자동 삭제 |
| ⑧ 그룹사 이동시 과거 회사 문서 조회 | 그룹사 형태로 이용 중인 경우에 표시되는 항목이며, 대표 회사만 설정 가능  - 그룹사 전적 사원(문서의 결재자, 공유자)이 과거 회사 문서를 조회할 수 있도록 설정, 해당 설정은 문서 생성 회사를 기준으로 함 - '사용'으로 설정 시 회사별 과거 문서 조회 여부 및 공유자 조회 허용 여부 설정   - 회사 별 과거 문서 조회를 '사용'으로 선택의 경우, 해당 회사에서 결재자인 문서를 전적 후의 회사에서도 조회 가능   - 회사별 공유자 조회를 '사용'으로 선택의 경우, 해당 회사에서 공유 대상인 문서를 전적 후의 회사에서도 조회 가능 |
| ⑩ 저장 | 클릭하여 설정한 내용 저장 |

- 선결재  

- 순차 결재  

- 지정 반려  

- 보류  

# 명칭 관리

| 영역 | 설명 |
| --- | --- |
| ① 결재 구분 명칭 설정 | 서식별 결재 구분 명칭 설정을 '사용'으로 선택하면 서식별 결재 구분 명칭 설정 가능, '사용 안 함'으로 선택하면 서식 전체에 공통으로 적용  - 공통 결재 구분 명칭 선택 |
| ② 처리 버튼 명칭 설정 | 서식별 처리 구분 명칭 설정을 '사용'으로 선택하면 서식별 처리 구분 명칭 설정 가능, '사용 안 함'으로 선택하면 서식 전체에 공통으로 적용  - 공통 처리 구분 명칭 선택 |
| ③ 작성 버튼 명칭 설정 | 서식별 작성 버튼 명칭 설정을 '사용'으로 선택하면 서식별 작성 버튼 명칭 설정 가능, '사용 안 함'으로 선택하면 서식 전체에 공통으로 적용  - 공통 작성 버튼 명칭 선택 |
| ④ 저장 | 클릭하여 설정한 내용 저장 |

# 알림 관리

사용자가 수신할 결재 관련 알림을 설정할 수 있습니다. 서비스 알림,  메일로 수신할 수 있으며 외부 솔루션(카카오톡, LINE, Slack, Google Chat 등)과 연동하여 알림 메시지를 수신할 수 있습니다.

외부 솔루션과의 알림 연동은 [서비스 연동 관리](/ko/admin-guides/manage-service/support-management/settings/#i-2)를 참고해 주십시오.

알림 관리 화면은 다음과 같이 구성되어 있습니다.

| 영역 | 설명 |
| --- | --- |
| ① 알림 채널 | 서비스 알림, 메일 형식의 알림 설정  - 외부 솔루션 연동 시 외부 솔루션명 표시 및 채널별 알림 설정 - '관리' 알림은 수신할 대상자를 추가, 메일로만 알림 수신(['유효하지 않은 결재 변경 요청 메일'](/#i-3-1) 참조)   - 사원, 부서 또는 그룹으로 수신 대상 설정     - 사원: 선택한 사원에게 알림 전송('인사 > 사원 관리' 목록에 있는 사원을 선택 가능)     - 부서: 선택한 부서 구성원에게 알림 전송     - 그룹: 선택한 그룹에 알림 전송 |
| ③ 저장 | 설정한 내용 저장 |

## 유효하지 않은 결재 변경 요청 메일

유효하지 않은 결재자 또는 수신처가 발생하면, 관리자에게 변경 요청 알림이 전송됩니다.

유효하지 않은 결재자/수신처 변경 요청 메일은 아래와 같이 구성되어 있습니다.

- '서식 바로가기'를 클릭하면 해당 서식의 결재자 또는 수신처를 바로 변경할 수 있습니다.
- '문서 바로가기'를 클릭하면 해당 문서의 조회 화면이 열립니다. 문서 내용 및 결재 현황을 확인할 수 있습니다.
- 하단의 '유효하지 않은 결재 관리'를 클릭하여 문서별 결재자, 수신처를 변경할 수 있습니다. 자세한 내용은 '[유효하지 않은 결재 관리](/ko/admin-guides/manage-service/manage-approval/document-authority/#i-1)'를 참고해 주세요.

문서 번호 관리 화면은 다음과 같이 구성되어 있습니다.

| 영역 | 설명 |

| ③ 서식 구분명 | 서식을 구분하기 위한 구분명을 서식별로 입력 |

| ⑤ 구분자 | 사용할 구분자 선택 |

# 데이터 불러오기

외부 데이터를 불러오는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. '템플릿 파일 다운로드'를 클릭합니다.
5. 다운로드한 템플릿 내의 가이드에 따라 엑셀 파일을 작성합니다.
     - 전체 공유 또는 일부 공유가 가능합니다.
     - 일부 공유는 사원/부서 단위로, 특정 사원 또는 특정 부서에 공유할 수 있습니다.
     - 특정 사원에게만 공유하려면 '공유 사원' 열에 사원의 업무 메일 주소를 입력합니다. 여러 개인 경우 콤마(,)로 구분합니다.
     - 특정 부서에게만 공유하려면, '공유 부서' 열에 부서 코드 및 하위 부서 포함 여부, 소속된 구성원 개별 입력 여부를 입력합니다. 입력 형식은 '부서 코드^하위 부서 포함 여부^소속된 구성원 개별 입력 여부'입니다.
       - 하위 부서 포함 여부, 소속된 구성원 개별 입력 여부는 각각 'Y' 또는 'N'으로 입력하며, 값 사이에 '^'로 구분합니다.
         - 예: 회계팀에 하위 부서를 포함하고 소속된 구성원 개별 입력으로 공유인 경우, '220000000456394^Y^Y'를 입력합니다. '220000000456394'만 입력하면 하위 조직을 포함하지 않고, 소속된 구성원 개별이 아닌 부서 단위로 공유됩니다.
     - 그룹사에 공유하려면 '공유 회사' 열에 공유할 회사의 연동키를 입력해 주십시오.
       - 공유할 회사 연동키를 입력하지 않으면 전체 그룹사에 공유될 수 있습니다.
6. STEP 2의 '파일 업로드'를 클릭한 후 저장한 템플릿 파일을 업로드합니다.
   - 업로드한 결과가 표시됩니다.
   - 입력한 내용 오류가 있을 경우, 오류가 발생한 곳이 빨간색으로 표시되며 하단에 오류 내용을 안내하는 메시지가 노출됩니다. 이 경우 엑셀 파일의 오류를 수정한 후 다시 업로드해 주십시오.
7. STEP 3에 업로드할 외부 문서를 준비합니다.  

   - 외부 결재 문서를 PDF 파일로 저장합니다. 파일명은 문서 번호와 동일해야 합니다.
   - 첨부 파일이 있는 문서의 경우, 문서 번호 이름으로 폴더를 생성한 후 첨부 파일을 모두 넣어 주십시오.
   - 업로드할 PDF파일 및 첨부 파일 폴더를 전체 선택 후, 마우스 오른쪽 버튼을 클릭하여 ZIP 파일로 압축합니다.  

     Mac OS에서는 압축 전용 프로그램을 사용하셔야 정상적으로 업로드 가능합니다.
8. STEP 3의 'PDF/첨부 파일 업로드'를 클릭한 후 준비한 ZIP 파일을 업로드합니다.  

9. 오류가 없다면 '저장' 버튼이 활성화되며, 저장 버튼 클릭 전에 '다운로드'를 클릭하여 업로드 문서 목록을 엑셀 파일로 다운로드할 수 있습니다.
   - PDF나 첨부 파일이 없더라도 저장은 가능하지만, 첨부가 누락된 문서가 있는지 확인하여 필요한 파일을 업로드해 주세요.
10. '저장'을 클릭한 후 확인 팝업창에서 '불러오기 내역으로 이동'을 클릭합니다.
    - 불러오기 내역 탭을 클릭하여 업로드한 내역을 확인할 수 있습니다.
    - 등록이 완료되면 신청자와 알림 관리의 수신 대상으로 설정된 사람에게 메일이 발송됩니다.

# 연동 코드 관리

컴포넌트에 연동하여 사용할 수 있는 연동 코드를 등록하고 관리할 수 있습니다. 등록된 연동 코드는 서식의 셀렉트 박스 내의 선택 항목으로 사용하거나, 고객사에서 사용 중인 기업정보시스템(Lagecy)와 연동하여 활용할 수 있습니다.

연동 코드 관리 화면은 다음과 같이 구성되어 있습니다.  

| 영역 | 설명 |
| --- | --- |
| ① 추가 | 연동할 컴포넌트 코드(항목)를 추가([연동 코드 추가](#i-6-1) 참고) |
| ② 연동 코드 목록 | 등록된 연동 컴포넌트 코드(항목), 를 클릭하여 연동 코드 수정([연동 코드 수정](#i-6-2) 참고) |
| ③ 추가 | 해당 코드에 상세 항목 등록([연동 코드 추가](#i-6-1) 참고) |
| ④ 업로드 | 코드 상세 항목과 항목 ID 일괄 업로드([연동 코드 업로드](#i-6-4) 참고) |
| ⑤ 다운로드 | 해당 코드의 코드 상세 항목과 항목 ID 다운로드([연동 코드 다운로드](#i-12) 참고) |
| ⑥ 세부 코드 목록 | 연동 컴포넌트 코드의 상세 항목  세부 코드 노출 순서 변경, 세부 코드 사용 여부 변경([세부 코드 사용 여부 변경](#i-6-3) 참고) |
| ⑦ 저장 | 등록 및 수정 내용 저장 |

## 연동 코드 추가

연동할 컴포넌트 코드(항목)를 추가하는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. 연동 코드 상단의  '추가'를 클릭합니다.
   - 연동 코드 추가 팝업창에 코드명과 코드 key를 입력하고 사용 여부를 선택합니다.
   - '저장'을 클릭합니다.
5. 등록한 연동 코드를 선택합니다.
6. 세부 코드 상단의 '추가'를 클릭합니다.
   - 추가된 행에 세부 코드, 세부 코드명을 입력하고 사용 여부를 선택합니다.
   - 세부 코드를 여러 개 등록하려면 이 과정을 반복합니다.
   - 순서 변경 아이콘을 클릭하여 표시 순서를 변경할 수 있습니다.
7. '저장'을 클릭합니다.

등록한 연동 코드를 수정하는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. 연동 코드 목록에서 변경할 연동 코드의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_modify-1.png)를 클릭합니다
   - 필요한 경우 연동 코드의 코드명, 사용 여부를 변경합니다.
   - 세부 코드를 수정하려면 세부 코드의 세부 코드명, 세부 코드명을 클릭 후 수정합니다.
   - 연동 코드별 세부 코드의 사용 여부를 변경할 수 있습니다.([세부 코드 사용 여부 변경](#i-6-3) 참고)
5. '저장'을 클릭합니다.

- 코드 Key는 수정할 수 없습니다.
- 사용 여부를 '사용 안 함'으로 변경하면 서식의 컴포넌트에 연동 코드 적용 시 해당 연동 코드가 조회되지 않습니다.([연동 코드 적용](/ko/admin-guides/manage-service/manage-approval/manage-forms/#i-1-9) 참고) 세부 코드의 사용 여부를 '사용 안 함'으로 변경하면 사용자가 서식 작성 시 해당 세부 코드가 조회되지 않습니다.
- 문서 내용에 포함된 연동 코드 세부 항목이 '사용 안 함'으로 변경 시 결재자는 진행 중 문서 내용을 수정할 수 없습니다.([문서 결재 > 문서 수정 후 동의](/ko/use-guides/approval/pc-web/approve-document/#i-3-7) 참고)

## 세부 코드 사용 여부 변경

연동 코드의 세부 코드 사용 여부를 변경하는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. 연동 코드 목록에서 연동 코드를 선택합니다.
5. 변경할 세부 코드에 마우스를 올린 후 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_open.png)를 클릭하여 사용 여부를 선택합니다.
6. '저장'을 클릭합니다.

서식에 적용되어 있는 연동 코드의 경우, 최소 1개의 세부 항목이 '사용' 상태여야 합니다.

## 연동 코드 업로드

연동 코드를 업로드하는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. 연동 코드 항목에서 업로드 대상 연동 코드를 선택합니다.
5. '업로드'를 클릭합니다.
6. 업로드 팝업창에서 '샘플 파일 다운로드'를 클릭해 샘플 파일을 다운로드합니다.
   - 샘플 파일이 엑셀 파일로 다운로드됩니다.
7. 샘플 파일을 참고하여 연동 코드를 작성한 후 저장합니다 .
8. 업로드 팝업창에서 '파일 첨부'를 클릭한 후 연동 코드 파일을 선택한 다음 '저장'을 클릭합니다.
9. 알림 팝업창에서 내용을 확인한 후 '확인'을 클릭합니다.
10. 확인 팝업창에서 내용을 확인한 후 '확인'을 클릭합니다.

- 업로드 실패 시 오류 원인을 해결한 후 다시 업로드해 주십시오.
- 업로드가 정상적으로 완료되면 해당 연동 코드 항목에서 코드 상세 항목과 항목 ID 정보를 확인할 수 있습니다.

## 연동 코드 다운로드

연동 코드를 다운로드하는 방법은 다음과 같습니다.

1. PC 웹 화면 오른쪽 상단의 '관리자 메뉴'를 클릭 후, 팝업에서 '결재 관리'를 클릭합니다.
2. 화면 왼쪽 상단의 ![](https://gsaas-help.pstatic.net/wp-content/uploads/2023/12/workplace_mmenu.png)을 클릭합니다.
4. 연동 코드 항목에서 다운로드 대상 연동 코드를 선택합니다.
5. '다운로드'를 클릭합니다.
   - 연동 코드 목록이 엑셀 파일로 다운로드됩니다.

다운로드된 파일은 `Downloads` 폴더에 `interlockcode.xlsx` 파일명으로 저장됩니다.

공공용 네이버웍스에서는 일부 기능이 제한됩니다. 자세한 내용은 [네이버웍스 기능 소개서](https://gov-naverworks.com/naver-works-function-list/)를 확인해주세요.
