# 보관 정책

이 기능은 아카이빙 옵션 상품 신청 후 사용 가능합니다.

아카이빙의 데이터 보관 기간과 대상을 설정합니다.

사용자가 데이터를 삭제하더라도 설정된 보관 기간과 대상에 해당하는 데이터가 보관되며, 이슈를 생성해 검색할 수 있습니다.  
단, 메일의 경우, 아카이빙 전 휴지통에서 영구 삭제된 메일은 보관되지 않습니다.

관리자의 설정에 따라 주고받은 메일, 메시지의 데이터를 1년에서 최대 10년까지 보관할 수 있습니다.

아카이빙 보관 정책의 기본 설정은 아래와 같습니다.

- 아카이빙 범위: 전체 도메인
- 보관 기간: 10년

단, 삭제한 구성원의 메일 데이터는 아카이빙할 수 없습니다.  
삭제한 구성원의 메시지 데이터는 아카이빙 보관 기간 동안 보관되며, 네이버웍스 해지 시 함께 삭제됩니다.

# 메시지 아카이빙

메시지 데이터를 보관하는 범위와 기간을 설정합니다.

1. Admin 왼쪽 메뉴에서 '아카이빙'을 선택해 메뉴를 펼친 후 '보관 정책'을 눌러 '보관 정책' 화면으로 이동합니다.
2. '메시지'를 선택합니다.
   - 아카이빙 범위: 메시지 데이터를 보관할 범위를 선택합니다. 전체 도메인, 특정 구성원, 사용 안 함 가운데 선택할 수 있습니다.
     - 전체 도메인: 회사 전체 구성원이 주고 받은 메시지 데이터를 보관합니다.
     - 특정 구성원: 특정 구성원이 주고 받은 메시지 데이터를 보관합니다. '특정 구성원'을 선택하면 구성원을 검색할 수 있는 '특정 구성원' 항목이 나타납니다. 선택할 수 있는 구성원 수의 제한은 없습니다.
     - 사용 안 함: 메시지 데이터를 보관하지 않습니다.
   - 보관기간: 메시지 데이터를 보관할 기간입니다. 1년 단위로 최장 10년까지 설정할 수 있습니다. 메시지를 보낸 시각, 받은 시각을 기준으로 설정한 기간 동안 보관합니다.  
     아카이빙을 신규 설정 시, 보관 기간을 기준으로 6개월 이전 과거 데이터부터 보관됩니다. 이후 아카이빙을 계속 사용할 때, 설정한 보관 기간이 지나면 *보관 기간*보다 이전의 데이터는 삭제됩니다.
3. '저장'을 누릅니다.

단, 외부 URL 타입의 Bot 메시지(Bot\_Audio, Bot\_Video, Bot\_Image)의 경우, 대화 내용만 보관되며, 첨부파일은 보관되지 않습니다.

# 메일 아카이빙

메일 데이터를 보관하는 범위와 기간을 설정합니다.

1. Admin 왼쪽 메뉴에서 '아카이빙'을 선택해 메뉴를 펼친 후 '보관 정책'을 눌러 '보관 정책' 화면으로 이동합니다.
2. '메일'을 선택합니다.
   - 아카이빙 범위: 메일 데이터를 보관할 범위를 선택합니다. 전체 도메인, 특정 구성원, 사용 안 함 가운데 선택할 수 있습니다.
     - 전체 도메인: 회사 전체의 메일 데이터를 보관합니다.
     - 특정 구성원: 특정 구성원의 메일함 데이터를 보관합니다. '특정 구성원'을 선택하면 구성원을 검색할 수 있는 '특정 구성원' 항목이 나타납니다. 선택할 수 있는 구성원 수의 제한은 없습니다.
     - 사용 안 함: 메일함 데이터를 보관하지 않습니다.
   - 보관기간: 메일 데이터를 보관할 기간입니다. 1년 단위로 최장 10년까지 설정할 수 있습니다. 메일을 보낸 시각, 받은 시각을 기준으로 설정한 기간 동안 보관합니다.  
     예를 들어, 2022년 1월 1일에 보관 기간을 1년으로 설정하면 2021년 1월 2일부터 2022년 1월 1일까지의 데이터가 보관됩니다.  
     관리자 설정에 따라 메일의 송수신 데이터를 보관할 수 있으며, 설정한 아카이빙 보관 기간을 기준으로 과거 데이터가 보관됩니다.  
     보관 처리를 완료하면 알림 메일을 전달해 드립니다. 보관할 데이터가 많으면 수일 동안 아카이빙이 진행될 수 있습니다.
3. '저장'을 누릅니다.

공공용 네이버웍스에서는 일부 기능이 제한됩니다. 자세한 내용은 [네이버웍스 기능 소개서](https://gov-naverworks.com/naver-works-function-list/)를 확인해주세요.
